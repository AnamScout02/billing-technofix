Pendahuluan & Fitur Sistem

Apa itu TechnoFix-Bill dan apa yang bisa dilakukan

ℹ️ Tentang TechnoFix-Bill

TechnoFix-Bill adalah sistem ISP Billing & Monitoring berbasis web yang dirancang untuk membantu operator internet skala kecil hingga menengah mengelola seluruh operasional jaringan dari satu platform terpadu.

Sistem ini bersifat multi-tenant — artinya banyak ISP dapat menggunakan satu server bersama, namun data masing-masing ISP sepenuhnya terisolasi satu sama lain.

Login & Navigasi Awal

Cara masuk ke sistem dan mengenal tampilan dashboard

🔐 Login ke Sistem

  1. Buka browser (Chrome/Firefox) dan masukkan alamat: http://[alamat-server]/app/frontend/auth/auth.html
  2. Masukkan Username dan Password yang diberikan admin
  3. Klik tombol Masuk
  4. Sistem akan otomatis mengarahkan ke halaman Dashboard
💡 Login pertama menggunakan akun Owner. Username default biasanya admin atau sesuai yang didaftarkan saat setup awal. Hubungi pengelola platform jika lupa akun.
⚠️ Jangan bagikan akun Owner kepada sembarang orang. Gunakan fitur Manajemen User untuk membuat akun terpisah untuk tim.

Setup Perangkat: MikroTik

Menambahkan dan menghubungkan router MikroTik ke sistem

🔌 Persiapan MikroTik

Sebelum menambahkan MikroTik ke sistem, pastikan:

  1. MikroTik sudah terhubung ke jaringan yang sama dengan server TechnoFix-Bill
  2. Fitur API di MikroTik sudah diaktifkan: IP → Services → api (port 8728)
  3. Catat IP address, username, dan password login MikroTik
  4. Pastikan tidak ada firewall yang memblokir port 8728

➕ Menambahkan MikroTik

  1. Klik menu Perangkat → pilih MikroTik dari bottom sheet
  2. Klik tombol Tambah MikroTik
  3. Isi form berikut:
  • Nama Perangkat *Nama pengenal, contoh: MikroTik Sangkur
  • IP Address *IP lokal MikroTik, contoh: 192.168.1.1
  • Port APIDefault: 8728 (biarkan jika tidak diubah)
  • Username *Username login MikroTik (biasanya admin)
  • Password *Password MikroTik (bisa kosong jika tidak ada password)
  • KoordinatLatitude, Longitude untuk tampil di Maps (opsional)
  1. Klik Simpan & Tes Koneksi
  2. Jika berhasil, status berubah menjadi ● Connected
⚠️ Jika muncul pesan "Gagal terhubung", periksa: IP address benar, API service aktif di MikroTik, dan tidak ada firewall/ACL yang memblokir.

🔄 Sinkronisasi Data

Setelah MikroTik terhubung, Anda bisa sinkronisasi data:

  • Sync PPPoE SecretAmbil daftar pelanggan dari MikroTik ke database lokal
  • Sync ProfileAmbil daftar profile PPPoE (paket kecepatan) dari MikroTik
  • Status RealtimePantau siapa yang sedang online langsung dari dashboard

✏️ Edit & Hapus MikroTik

Di halaman daftar MikroTik, setiap baris memiliki tombol aksi:

TombolFungsi
✏️ EditUbah nama, IP, username, atau password
🔄 Test KoneksiTes ulang koneksi ke perangkat
🗑️ HapusHapus perangkat dari sistem (data pelanggan tidak ikut terhapus)

Setup Perangkat: OLT

Optical Line Terminal — monitoring jaringan fiber optik

📡 Menambahkan OLT

  1. Klik menu Perangkat → pilih OLT
  2. Klik Tambah OLT
  3. Isi data OLT:
  • Nama OLT *Contoh: OLT ZTE-01 Kantor
  • Tipe OLT *Pilih vendor: ZTE / Huawei / FiberHome / Lainnya
  • IP Address *IP manajemen OLT
  • Port SSH/TelnetDefault: 22 untuk SSH
  • Username / Password *Credential login OLT
  • Jumlah PON PortTotal port GPON yang tersedia
  • KoordinatLokasi OLT untuk ditampilkan di Maps
💡 Koordinat bisa diisi manual (copy dari Google Maps) atau klik tombol Deteksi Lokasi jika Anda sedang berada di lokasi OLT.

📊 Monitoring ONU

Setelah OLT tersambung, sistem dapat membaca data ONU secara realtime:

  • RX PowerKekuatan sinyal yang diterima ONU (normal: -8 s/d -27 dBm)
  • TX PowerKekuatan sinyal yang dipancarkan ONU
  • Status ONUOnline / Offline / Loss of Signal
  • Serial NumberIdentitas unik modem pelanggan
  • Slot/PortPosisi fisik ONU di dalam OLT (misal: 1/1/1)
Indikator SinyalNilaiKeterangan
● Normal> -20 dBmSinyal bagus, koneksi stabil
● Sedang-20 s/d -27 dBmPerlu diperiksa kabel/konektor
● Lemah< -27 dBmSegera periksa splitter/kabel

🔍 Scan ONU Baru

Untuk mendeteksi ONU yang baru dipasang namun belum terdaftar:

  1. Buka halaman OLT → klik Scan ONU Baru
  2. Sistem akan membaca ONU unconfigured dari OLT
  3. Serial Number ONU baru akan muncul dalam daftar
  4. Gunakan SN ini saat menambahkan data pelanggan baru

Manajemen ODC

Optical Distribution Cabinet — titik distribusi fiber ke area

🗄️ Apa itu ODC?

ODC (Optical Distribution Cabinet) adalah kotak distribusi kabel fiber optik yang dipasang di area tertentu. ODC menerima sinyal dari OLT dan mendistribusikannya ke beberapa ODP di sekitarnya. Dalam topologi jaringan: OLT → ODC → ODP → Pelanggan

➕ Menambahkan ODC

  1. Klik menu Perangkat → pilih ODC
  2. Klik tombol Tambah ODC
  3. Isi form berikut:
  • Nama ODC *Contoh: ODC-A Perumahan Griya
  • OLT Induk *Pilih OLT yang terhubung ke ODC ini
  • Jumlah PortTotal port output ODC (biasanya 8 atau 16)
  • Lokasi / AlamatDeskripsi lokasi fisik ODC
  • KoordinatLatitude, Longitude untuk tampil di Maps
  • KeteranganCatatan tambahan (opsional)
  1. Klik Simpan

✏️ Edit ODC

  1. Di daftar ODC, klik ikon Edit (✏️) pada baris ODC yang ingin diubah
  2. Ubah data yang diperlukan
  3. Klik Simpan Perubahan
📌 Mengubah OLT induk atau jumlah port pada ODC yang sudah memiliki ODP terhubung dapat mempengaruhi topologi di Maps.

📋 Melihat Port ODC

Klik nama ODC untuk melihat detail penggunaan port:

  • Port TerpakaiPort yang sudah terhubung ke ODP
  • Port KosongPort yang masih tersedia
  • ODP TerhubungDaftar ODP yang terhubung ke port ODC ini

Manajemen ODP

Optical Distribution Point — titik distribusi ke rumah pelanggan

🔆 Apa itu ODP?

ODP (Optical Distribution Point) adalah titik akhir distribusi fiber sebelum ke rumah pelanggan. Biasanya dipasang di tiang atau kotak di dekat cluster perumahan. Setiap ODP memiliki beberapa port (4, 8, atau 16 port) untuk menghubungkan pelanggan.

➕ Menambahkan ODP

  1. Klik menu Perangkat → pilih ODP
  2. Klik tombol Tambah ODP
  3. Isi form:
  • Nama ODP *Contoh: ODP-01 Jl. Merdeka
  • ODC Induk *Pilih ODC yang terhubung
  • Port ODCNomor port di ODC yang digunakan ODP ini
  • Jumlah Port ODPKapasitas port untuk pelanggan (4/8/16)
  • Lokasi / Alamat *Alamat fisik ODP di lapangan
  • KoordinatLatitude, Longitude untuk Maps
  1. Klik Simpan

👁️ Monitoring Port ODP

Di halaman detail ODP, Anda bisa melihat:

InfoKeterangan
Port TerisiPort yang sudah ada pelanggan
Port KosongPort tersedia untuk pelanggan baru
Pelanggan per PortNama pelanggan yang terhubung di masing-masing port
Status ONUSinyal modem pelanggan (dari data OLT)
💡 Saat menambah pelanggan baru, pilih ODP terlebih dahulu — sistem akan otomatis menampilkan port yang masih kosong.

Profile PPPoE & Paket Kecepatan

Kelola paket internet dan harga langsung dari MikroTik

⚡ Apa itu Profile PPPoE?

Profile PPPoE adalah paket kecepatan internet yang tersimpan di MikroTik. Setiap profile mendefinisikan batas download dan upload untuk pelanggan yang menggunakannya. Contoh: Profile 10M/5M = download 10 Mbps, upload 5 Mbps.

🔎 Membuka Halaman Profile PPPoE

  1. Klik menu Perangkat → pilih Profile PPPoE
  2. Pilih perangkat MikroTik dari dropdown di bagian atas
  3. Daftar profile akan muncul secara otomatis

➕ Menambah Profile Baru

  1. Klik tombol Tambah Profile
  2. Isi form:
  • Nama Profile *Nama unik, contoh: 10M atau PAKET-RUMAHAN
  • Download *Kecepatan download + satuan (K/M/G)
  • Upload *Kecepatan upload + satuan (K/M/G)
  • Harga/BulanHarga tagihan per bulan untuk pelanggan paket ini
  • Catatan BandwidthKeterangan tambahan misal: Bonus 20M jam 00-06
  1. Klik Tambah Profile — sistem otomatis membuat profile di MikroTik
💡 Profile langsung tersimpan ke MikroTik secara realtime. Tidak perlu login ke Winbox untuk membuat profile baru.

💲 Mengatur Harga Profile

Harga bisa diatur langsung di tabel tanpa membuka form edit:

  1. Di kolom Harga/Bulan, klik pada nilai harga yang ada
  2. Nilai berubah menjadi input — ketik harga baru
  3. Tekan Enter atau klik di luar kolom untuk menyimpan
📌 Harga profile digunakan sebagai default saat menambah pelanggan baru dengan paket tersebut.

Manajemen Pelanggan

Tambah, edit, monitor, dan kelola seluruh pelanggan ISP

➕ Menambah Pelanggan Baru

  1. Buka menu Pelanggan
  2. Pilih MikroTik dari dropdown perangkat
  3. Klik tombol Tambah Pelanggan
  4. Isi form data pelanggan:
  • Nama Lengkap *Nama pelanggan sesuai KTP
  • Username PPPoE *Username untuk koneksi internet (unik, huruf kecil)
  • Password PPPoE *Password koneksi internet
  • No. Telepon *Digunakan untuk login portal dan notifikasi WA
  • Profil Paket *Pilih profile PPPoE yang akan digunakan
  • AlamatAlamat lengkap pelanggan
  • ODP *Pilih ODP tempat pelanggan terhubung
  • Port ODPNomor port ODP yang digunakan (otomatis tampil port kosong)
  • Serial NumberSN modem/ONU pelanggan
  • Tanggal PasangTanggal aktivasi layanan
  • Tanggal Jatuh TempoTanggal tagihan berakhir tiap bulan
  • KolektorPilih kolektor yang bertanggung jawab menagih pelanggan ini
  1. Klik Simpan — pelanggan otomatis ditambahkan ke MikroTik

👁️ Monitor Status Pelanggan

Di daftar pelanggan, setiap baris menampilkan status realtime:

StatusKeteranganWarna
OnlinePelanggan sedang terhubung ke internet● Online
OfflineTidak ada koneksi aktif saat ini● Offline
TerisolirAkun dinonaktifkan (belum bayar)● Terisolir
💡 Klik tombol Refresh (🔄) di header untuk memperbarui status semua pelanggan dari MikroTik.

✏️ Edit Data Pelanggan

  1. Klik ikon Edit (✏️) atau klik nama pelanggan untuk buka detail
  2. Di halaman detail, klik tombol Edit
  3. Ubah data yang perlu diperbarui
  4. Klik Simpan Perubahan
⚠️ Mengubah Username PPPoE akan otomatis mengupdate di MikroTik. Pastikan pelanggan memasukkan ulang username baru di router/modem mereka.

🔍 Filter & Pencarian

  • Cari Nama/UsernameKetik di kolom pencarian untuk filter pelanggan
  • Filter StatusTampilkan semua / online saja / offline saja / terisolir
  • Filter Jatuh TempoTampilkan pelanggan yang jatuh tempo bulan ini
  • Filter KolektorTampilkan pelanggan milik kolektor tertentu

⛔ Isolir & Aktifkan Kembali

Isolir digunakan untuk menangguhkan koneksi pelanggan yang belum membayar:

  1. Buka detail pelanggan atau cari di daftar
  2. Klik tombol Isolir — koneksi pelanggan langsung diputus di MikroTik
  3. Setelah pelanggan membayar, klik Aktifkan untuk memulihkan koneksi

📋 Riwayat Transaksi Pelanggan

Di halaman detail pelanggan, scroll ke bawah untuk melihat:

  • Riwayat BayarDaftar pembayaran yang pernah dilakukan
  • Status TagihanTagihan bulan berjalan dan bulan sebelumnya
  • Data ONU/ModemSinyal RX/TX dari OLT (jika menggunakan fiber)

Keuangan, Tagihan & Loket

Catat pembayaran, kelola tagihan, dan terima bayar di loket

🧾 Tagihan Pelanggan

Tagihan dibuat otomatis berdasarkan tanggal jatuh tempo pelanggan. Untuk mengakses:

  1. Klik menu Keuangan → pilih Tagihan
  2. Daftar tagihan semua pelanggan akan tampil
  3. Filter berdasarkan: status (Lunas/Belum Lunas), bulan, atau nama pelanggan
Status TagihanKeterangan
LunasTagihan sudah dibayar
Belum LunasTagihan belum dibayar, mungkin perlu diisolir
SebagianBaru dibayar sebagian

💵 Mencatat Pembayaran

  1. Di daftar tagihan atau detail pelanggan, klik Bayar
  2. Isi jumlah yang dibayar
  3. Pilih metode pembayaran (Tunai / Transfer / QRIS)
  4. Tambahkan catatan jika perlu
  5. Klik Simpan Pembayaran
  6. Sistem otomatis memperbarui status tagihan dan mencatat ke laporan keuangan

🏪 Loket / Kasir

Fitur Loket dirancang untuk penerimaan pembayaran cepat:

  1. Klik menu Keuangan → pilih Loket
  2. Cari nama atau username pelanggan di kolom pencarian
  3. Klik pelanggan yang akan membayar
  4. Pilih bulan yang dibayarkan (bisa lebih dari 1 bulan)
  5. Sistem menghitung total otomatis
  6. Klik Proses Pembayaran
  7. Struk/tanda terima bisa dicetak atau di-screenshot
💡 Kolektor dapat mengakses Loket untuk mencatat pembayaran yang diterima di lapangan dan otomatis terupdate ke sistem pusat.

📊 Laporan Keuangan

  1. Klik menu Keuangan → pilih Keuangan
  2. Pilih rentang bulan untuk melihat laporan

Laporan mencakup:

  • Total PemasukanJumlah pembayaran yang diterima dalam periode
  • Tagihan LunasJumlah pelanggan yang sudah membayar
  • Tagihan Belum LunasJumlah piutang yang belum diterima
  • Detail TransaksiDaftar semua pembayaran dengan nama kasir/kolektor

💳 Bayar Online (Midtrans/Xendit)

Jika sudah diintegrasikan dengan payment gateway:

  1. Klik menu Keuangan → pilih Bayar Online
  2. Pelanggan akan mendapatkan link pembayaran
  3. Pembayaran via QRIS, transfer bank, kartu kredit
  4. Status tagihan terupdate otomatis setelah pembayaran berhasil
📌 Fitur bayar online membutuhkan konfigurasi API key dari Midtrans atau Xendit. Klik tombol Atur Gateway di halaman Bayar Online untuk mengisinya.

Notifikasi WhatsApp

Kirim pengingat tagihan otomatis ke pelanggan via WhatsApp

📲 Cara Menggunakan Notifikasi WA

  1. Klik menu Keuangan → pilih Notifikasi WA
  2. Pilih periode (bulan) tagihan di toolbar atas
  3. Klik Kirim Pengingat — sistem mengirim pesan pengingat ke semua pelanggan dengan tagihan belum lunas pada periode tersebut
  4. Riwayat pengiriman akan muncul di tabel log (status: Terkirim / Mock / Gagal)
💡 Gunakan tombol Kirim Uji untuk mengirim pesan percobaan ke satu nomor sebelum melakukan blast ke semua pelanggan.

📝 Template Pesan

Template pengingat dapat diubah lewat Atur Gateway. Placeholder yang tersedia:

PlaceholderDiganti Dengan
{nama}Nama pelanggan
{nominal}Jumlah tagihan
{periode}Periode tagihan (bulan & tahun)
{jatuh_tempo}Tanggal jatuh tempo
{isp}Nama ISP
📌 Pesan dikirim ke nomor telepon yang tersimpan di data pelanggan. Pastikan nomor WA pelanggan sudah benar dan aktif.

⚙️ Konfigurasi WA Gateway

Klik tombol Atur Gateway di halaman Notifikasi WA (memerlukan akses Keuangan) untuk mengatur:

  • ProviderFonnte atau Wablas
  • URL GatewayOpsional, khusus untuk Wablas
  • Token APIToken autentikasi dari provider WA
  • Template PesanIsi pesan pengingat dengan placeholder di atas
  • Aktifkan Pengiriman LiveJika nonaktif, sistem berjalan dalam mode mock — pesan hanya dicatat di log tanpa benar-benar terkirim
  • Pengingat OtomatisJika aktif, sistem otomatis mengirim pengingat H-3, hari-H, dan H+3 (telat) untuk tiap tagihan, masing-masing maksimal sekali

Maps Topologi Jaringan

Visualisasi peta jaringan fiber dari OLT hingga pelanggan

🗺️ Membuka Maps

  1. Klik menu Maps di navigasi
  2. Peta akan memuat dan menampilkan semua perangkat yang sudah ada koordinatnya
  3. Gunakan scroll mouse untuk zoom in/out, klik-tahan untuk geser peta

📍 Marker di Peta

MarkerPerangkatWarna
📡 Antena BesarOLTBiru tua
🗄️ KotakODCBiru muda
🔆 Titik KecilODPHijau
🏠 RumahPelangganHijau (online) / Merah (offline)

Garis yang menghubungkan antar marker menunjukkan kabel fiber optik:

  • Garis Tebal BiruKoneksi OLT → ODC (backbone)
  • Garis Tipis BiruKoneksi ODC → ODP (distribusi)
  • Garis Tipis HijauKoneksi ODP → Pelanggan (last mile)

🖱️ Interaksi dengan Peta

  • Klik MarkerTampilkan popup informasi perangkat (nama, IP, status, sinyal)
  • Klik PelangganTampilkan nama, status online, sinyal ONU, dan tombol aksi cepat
  • Filter PanelTampilkan/sembunyikan layer: OLT, ODC, ODP, atau Pelanggan
  • FullscreenKlik ikon fullscreen untuk tampilan lebih luas

📌 Menambahkan Koordinat Perangkat

Agar perangkat muncul di Maps, koordinat harus diisi saat tambah/edit data:

  1. Buka edit perangkat (OLT/ODC/ODP/Pelanggan)
  2. Di kolom Koordinat, masukkan nilai latitude,longitude
  3. Contoh: -8.123456,114.234567
  4. Atau klik Deteksi Lokasi untuk mengisi otomatis dari GPS perangkat (hanya bekerja di HTTPS atau localhost)
💡 Cara mudah ambil koordinat: buka Google Maps → klik kanan di lokasi → salin koordinat yang muncul.

Portal Pelanggan

Halaman khusus pelanggan untuk cek status, tagihan, dan speed test

🌐 Akses Portal Pelanggan

Portal pelanggan dapat diakses di:

Alamat Portal
http://[alamat-server]/app/frontend/portal/portal_login.html

Portal ini terpisah dari sistem admin — khusus untuk pelanggan akhir.

🔑 Login Portal Pelanggan

  1. Buka alamat portal di browser atau bagikan link ke pelanggan
  2. Masukkan Username PPPoE (sama dengan username koneksi internet)
  3. Masukkan Nomor Telepon yang terdaftar (digunakan sebagai password)
  4. Klik Masuk
📌 Password portal = nomor telepon pelanggan yang tersimpan di data admin. Jika pelanggan lupa, minta admin memperbarui nomor di data pelanggan.

📤 Kirim Tiket Laporan

  2. Klik Buat Laporan Baru
  3. Pilih Kategori: Gangguan Internet / Kecepatan Lambat / Pertanyaan Tagihan / Lainnya
  4. Tulis deskripsi masalah dengan jelas
  5. Klik Kirim
  6. Tiket akan muncul di dashboard admin untuk ditangani teknisi

Manajemen User & Tim

Kelola akun pengguna sistem dengan hak akses berbeda-beda

👥 Membuka Manajemen User

  1. Klik ikon profil di sudut kanan atas
  2. Pilih Manajemen User dari dropdown
  3. Daftar semua user dalam workspace Anda akan muncul
